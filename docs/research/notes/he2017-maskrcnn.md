# Setting

Problem class: **instance segmentation** — simultaneous per-instance pixel masks, bounding boxes, and class labels for every object in an image. This is distinct from semantic segmentation (per-pixel class labels, no instance distinction) and from object detection (boxes only, no pixel masks).

Inputs: RGB image, typically resized so the shorter edge is 800 px (Section 3.1). Outputs: for each detected instance, a class label + confidence score + bounding box + binary mask in the bounding box. The binary mask is predicted at $m \times m$ resolution — $m = 14$ for the ResNet-C4 head, $m = 28$ for the FPN head (Figure 4) — then resized to the box size at inference (Section 3.1 Inference). Headline benchmark: COCO (80 object classes, train/val/test-dev splits). The paper also demonstrates person keypoint detection as a parallel secondary task using the same framework.

# Core idea

Mask R-CNN extends Faster R-CNN's two-stage pipeline (RPN for region proposals + RoI head for classification + box regression) by adding a **third parallel output branch** — a small FCN that predicts a per-class binary mask at each RoI (Section 3, §Mask R-CNN). The mask and class predictions are intentionally decoupled: the mask branch outputs $K$ independent binary masks (one per class), and the class label chosen by the classification head selects which mask to use. This avoids per-pixel softmax competition between classes.

**RoIAlign** replaces RoIPool's two coarse quantization steps with bilinear sampling at floating-point coordinates inside each RoI bin (Section 3, §RoIAlign). RoIPool quantizes the continuous coordinate $x$ to $[x/16]$ (rounding to the nearest feature-map cell at stride 16), then quantizes again when dividing into spatial bins. These misalignments are negligible for classification but catastrophic for pixel-accurate mask prediction. RoIAlign uses $x/16$ (no rounding) and computes exact feature values by bilinear interpolation at four regularly spaced sampling points per bin.

**Per-class sigmoid mask loss** $L_\text{mask}$: the mask branch produces a $Km^2$-dimensional output encoding $K$ binary masks of resolution $m \times m$. A per-pixel sigmoid is applied; $L_\text{mask}$ is the average binary cross-entropy loss computed only on the $k$-th mask channel (the ground-truth class channel). Masks for other classes do not contribute to the loss. Total multi-task loss per RoI: $L = L_\text{cls} + L_\text{box} + L_\text{mask}$ (Section 3, §Mask R-CNN).

The framework also extends naturally to **person keypoint detection**: model each of $K_\text{kp}$ joint types as a one-hot binary mask (one spatial channel per joint), apply softmax cross-entropy over spatial positions, with a single foreground pixel per keypoint channel (Section 5).

# Assumptions

1. **Hard**: A CNN backbone (ResNet-50, ResNet-101, ResNeXt-101) pre-trained on ImageNet is available. The mask branch is not trained from scratch; ImageNet initialisation is assumed throughout.
2. **Hard**: Faster R-CNN's two-stage architecture (RPN + RoI head) is the detection substrate. Mask R-CNN is a two-stage instance segmenter; it is not a one-stage or anchor-free detector.
3. **Hard**: RoIs are generated by an RPN trained on the same image (or jointly). The mask branch is trained via the multitask loss jointly with the classification and box-regression heads.
4. **Soft**: RoIAlign's bilinear sampling assumes the feature map is locally smooth. This holds in practice for both C4-only features and FPN pyramidal features, but could degrade with very high-stride networks or severe feature noise.
5. **Soft**: Mask resolution is $m \times m$ per RoI: $28 \times 28$ for FPN head, $14 \times 14$ for C4 head (Figure 4). Higher resolution improves fine-grained mask quality but increases memory and compute proportionally.
6. **Soft**: The mask branch is trained and evaluated per ground-truth class. For novel classes or open-vocabulary settings, the mask branch has no mechanism for generalisation without retraining.

# Failure regime

- **Small and thin instances**: the mask branch predicts at fixed $28 \times 28$ per RoI (FPN head; $14 \times 14$ for C4), then upsamples to the box size. For very small boxes, the effective mask resolution is low; thin elongated objects (wire, rope, antenna) can be poorly segmented.
- **Heavily overlapping same-class instances**: NMS during inference removes near-duplicate detections. If two instances of the same class heavily overlap, one detection is suppressed and its mask is never predicted — the framework can silently drop one of two closely overlapping objects of the same class.
- **Crowded, densely packed scenes**: per-RoI mask computation at FPN feature levels is fixed-scale; instances assigned to higher pyramid levels may be under-resolved. FCIS+++ exhibited systematic boundary artefacts on overlapping instances (Figure 6); Mask R-CNN avoids this but still struggles at high density.
- **Open-vocabulary / zero-shot classes**: the mask branch has $K$ output channels for the $K$ training classes. It cannot produce masks for unseen classes without retraining or adaptation.
- **Real-time or resource-constrained inference**: the model runs at approximately 5 fps on a Tesla M40 GPU (Abstract, Figure 5 caption); this is not suitable for mobile or edge deployment.

# Numerical sensitivity

- **RoIAlign vs. RoIPool**: Table 2c (Section 4.2 ablations) — RoIAlign (avg-pool variant) achieves AP = 30.3, AP$_{75}$ = 31.5 on ResNet-50-C4; RoIPool achieves AP = 26.9, AP$_{75}$ = 26.4. The improvement is $\sim$3 AP and $\sim$5 AP$_{75}$. RoIWarp (bilinear but with quantization) achieves AP = 27.2, confirming that alignment (not bilinear interpolation alone) is the decisive factor.
- **Per-class sigmoid vs. per-pixel softmax**: Table 2b (Section 4.2 ablations, ResNet-50-C4) — sigmoid achieves AP = 30.3 vs. softmax AP = 24.8; gain is +5.5 AP. The paper states this decoupling "is key for good instance segmentation results" (Section 3, §Mask R-CNN).
- **Mask resolution $m \times m$**: confirmed from Figure 4 — the **ResNet-C4 head** produces $14 \times 14 \times K$ masks; the **FPN head** produces $28 \times 28 \times K$ masks (and the FPN head outputs the $80 \cdot 28^2$ tensor explicitly visible in the figure annotation). Both values are also visible in the pdftotext extraction of Figure 4 (column labels "14×14 ... 14×14 ... 28×28 28×28"). The C4 variant's heavier res5 box head makes it ~400 ms/image vs ~195 ms/image for ResNet-101-FPN at inference (Section 4.4 §Detection Results / Inference).
- **Backbone vs. mAP** (Table 2a, minival; Table 1, test-dev):
  - ResNet-50-C4: minival mask AP = 30.3 (Table 2a)
  - ResNet-101-C4: minival mask AP = 32.7 (Table 2a); test-dev AP = 33.1 (Table 1)
  - ResNet-50-FPN: minival mask AP = 33.6 (Table 2a)
  - ResNet-101-FPN: minival mask AP = 35.4 (Table 2a); test-dev AP = 35.7 (Table 1)
  - ResNeXt-101-FPN: minival mask AP = 36.7 (Table 2a); test-dev AP = 37.1 (Table 1)
- **Training schedule** (Section 3.1, §Training): 8 GPUs, 2 images per GPU (effective mini-batch size 16), 160k iterations total, SGD with initial LR = 0.02 reduced by 10× at 120k iterations, weight decay = 0.0001, momentum = 0.9. For ResNeXt backbones: 1 image per GPU, same total iterations, starting LR = 0.01.
- **Inference speed**: approximately 5 fps on a Tesla M40 (Abstract; Figure 5 caption). Mask branch adds $\sim$20% overhead over Faster R-CNN counterpart (Section 3.1, §Inference).
- **Mask output binarisation**: the $m \times m$ floating-point mask output is resized to the RoI size, then binarized at threshold 0.5 (Section 3.1, §Inference).

# Applicability

- **Use when**: COCO-style instance segmentation (80 classes, closed vocabulary); multi-task training combining detection + segmentation + optional keypoints; well-resourced GPU training (8 GPUs typical); strong off-the-shelf baselines via Detectron2 (Apache-2.0); when pixel-accurate instance boundaries matter more than raw speed.
- **Don't use when**: dense panoptic segmentation at scale (use Panoptic FPN, Mask2Former, or DETR-derived methods); real-time mobile inference at <10 ms budgets (use YOLACT, SOLO, or YOLOv8-seg); semantic segmentation of scenes without instance separation needed (use FCN, DeepLab); when one-stage or anchor-free architecture is required for deployment constraints; open-vocabulary instance segmentation without fine-tuning.
- **Compared against** (Table 1): MNC (Dai et al. 2016, COCO 2015 segmentation challenge winner, AP = 24.6), FCIS +OHEM (Li et al. 2016, COCO 2016 winner, AP = 29.2), FCIS+++ +OHEM (AP = 33.6 — Mask R-CNN ResNet-101-FPN surpasses this at AP = 35.7 without test-time augmentation).

# Connections

- **Builds on**:
  - `long2015-fcn`: FCN-style per-pixel prediction forms the mask branch. Mask R-CNN's mask head is described as "a small FCN applied to each RoI" (Section 3, §Mask R-CNN and Abstract). The key departure from FCN practice is the use of per-pixel sigmoid + binary cross-entropy rather than softmax + multinomial cross-entropy.
  - Faster R-CNN (Ren et al. 2015, bib [36]): the two-stage detection substrate (RPN + RoI head). Mask R-CNN "adopts the same two-stage procedure" (Section 3).
  - FPN (Lin et al. 2017, bib [27]): multi-scale feature pyramid used as the default (and best-performing) backbone in ablations and main results.
- **Enables**: Cascade Mask R-CNN, HTC (Hybrid Task Cascade), Panoptic FPN, PointRend — all build directly on Mask R-CNN's framework. The Detectron / Detectron2 / mmdetection ecosystems adopted Mask R-CNN as a primary baseline.
- **Refutes / supersedes**: the MNC and FCIS lines of instance segmenters. Mask-from-detection with decoupled per-class binary masks became the dominant practical paradigm for two-stage instance segmentation for several years following publication.
